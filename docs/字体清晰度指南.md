# Gemini API 字体清晰度指南

> 本文档基于 Gemini 图像生成 API 的实测经验总结，帮助你在生成海报时获得最佳文字清晰度。

## 核心结论

**文字越少，字号越大，清晰度越高。**

---

## 一、为什么文字会不清晰？

### 1.1 API 本质限制

Gemini 图像生成 API 是基于像素生成的图像模型，不是矢量设计工具：

- **像素渲染**：文字是作为图像的一部分生成的，不是独立的文字层
- **中文复杂度**：中文字符笔画复杂（平均 10-15 笔），比英文（平均 3-5 字母）难渲染
- **优先级问题**：模型优先保证整体视觉效果，文字清晰度往往被牺牲

### 1.2 具体不清晰的情况

| 场景 | 示例 | 结果 |
|------|------|------|
| 单行过长 | `"12/22（周一）20:00 开营分享 - 经验拆解：大龄以为要失业，人工智能产品岗竟成我的人生破局点"` | ❌ 40+字，模糊 |
| 密度太高 | 4 行课程安排，每行 30+ 字 | ❌ 整体模糊 |
| 背景复杂 | 赛博朋克风 + 大量特效 + 小字 | ❌ 文字淹没在特效中 |
| 整体信息过多 | 7 个区域 + 每个区域都有文字 | ❌ 模型被迫压缩文字 |

---

## 二、什么样的情况下字会清晰？

### 2.1 黄金法则

**单行不超过 15-20 个中文字**

| 字数 | 清晰度 |
|------|--------|
| 10 字以内 | ✅ 非常清晰 |
| 10-15 字 | ✅ 清晰可读 |
| 15-20 字 | ⚠️ 基本清晰 |
| 20-25 字 | ⚠️ 开始模糊 |
| 25-35 字 | ❌ 难以辨认 |
| 35+ 字 | ❌ 完全不可读 |

### 2.2 实测对比

**不清晰的示例：**
```
❌ "12/22（周一）20:00 开营分享 - 经验拆解：大龄以为要失业，人工智能产品岗竟成我的人生破局点"
   (47 个字符)
```

**清晰的示例：**
```
✅ "活动期间：扫码即可参与"
   (10 个字符)

✅ "推荐一人得10%返现"
   (9 个字符)
```

### 2.3 清晰度影响因素（按重要性排序）

1. **文字长度** ⭐⭐⭐⭐⭐
   - 越短越好，单行 ≤ 15 字最佳

2. **文字密度** ⭐⭐⭐⭐
   - 一张图的文字区域越少越好
   - 建议 ≤ 4 个文字区域

3. **对比度** ⭐⭐⭐
   - 高对比度更清晰（白底黑字、黑底白字）
   - 避免渐变文字、半透明文字

4. **背景复杂度** ⭐⭐
   - 纯色背景 > 渐变背景 > 复杂场景背景
   - 文字背景加遮罩/卡片可提升清晰度

5. **字体复杂度** ⭐
   - 简单字体 > 装饰字体
   - 粗体 > 细体

## 三、实用解决方案

### 3.1 方案一：精简文案（推荐）

**原则：能用 5 个字说清楚，不要用 10 个字。**

| 原文案 | 优化后 |
|--------|--------|
| `"12/22（周一）20:00 开营分享 - 经验拆解：大龄以为要失业，人工智能产品岗竟成我的人生破局点"` | `"12/22 周一 20:00 开营分享"` |
| `"从0到1打造AI产品，手把手实战教学"` | `"从0到1打造AI产品"` |
| `"想转行的产品经理/运营人员"` | `"产品经理/运营"` |

### 3.2 方案二：分图生成（强烈推荐）

将一张长图拆分成多张短图，每张图信息量少，文字可以放得更大。

**示例：课程海报拆分**

| 图 | 内容 | 文字量 |
|---|------|--------|
| 图1 | 标题 + 副标题 + 视觉主体 | 少，大而清晰 |
| 图2 | 课程安排（4条） | 中等 |
| 图3 | 适合人群 + CTA + 二维码 | 少，大而清晰 |

用户可以手动拼接或在设计工具中组合。

### 3.3 方案三：后期加文字（最清晰）

**工作流：**
1. 提示词中明确说明 "不要生成文字" 或 "文字用占位符"
2. 生成纯视觉底图
3. 用 Canva/Figma/Photoshop 添加精确文字

**优点：** 文字完美清晰，可随时修改
**缺点：** 需要额外工具

## 四、提示词最佳实践

### 4.1 强调清晰度的提示词

```prompt
【最高优先级要求】所有文字必须超大、超清晰、毫不含糊！
文字清晰度优先于所有视觉效果！
每个字都必须清晰可读，单行不超过15个字！
```

### 4.2 文字区域描述

**好的描述：**
```
❌ "课程安排：12/22（周一）20:00 开营分享 - 经验拆解：..."

✅ "课程安排区域，超大超粗清晰文字：
   12/22 周一 20:00 开营分享
   12/23 周二 20:00 第一讲"
```

### 4.3 避免的描述

- ❌ "精美的排版" → 模型会追求美观而牺牲清晰度
- ❌ "紧凑的布局" → 会导致文字被压缩
- ❌ "丰富的内容" → 会导致信息过载

---

## 五、快速检查清单

生成前检查：

- [ ] 每行文字 ≤ 15 字
- [ ] 文字区域 ≤ 4 个
- [ ] 避免长句，用短语
- [ ] 提示词中强调 "超大超粗超清晰"
- [ ] 考虑是否需要分图生成

## 六、常见问题

### Q1: 为什么英文比中文清晰？

A: 英文字符结构简单（平均 3-5 个笔画），中文字符复杂（平均 10-15 个笔画）。

### Q2: 放大图片有用吗？

A: 没用。API 生成的是固定分辨率，放大只是像素插值，不会变清晰。

### Q3: 用其他 API 会更好吗？

A: 不同 API 有差异，但文字渲染是整个行业的共性问题：
- DALL-E 3: 中等
- Midjourney: 较差
- Stable Diffusion: 需要专用 ControlNet
- Gemini: 中等偏上

最可靠的方案始终是：**减少文字** 或 **后期添加文字**。

---

## 总结

| 关键点 | 记住这个 |
|--------|----------|
| 字数限制 | **单行 ≤ 15 字** |
| 最佳方案 | **分图生成 + 手动拼接** |
| 最清晰 | **后期添加文字** |
| 次优方案 | **精简文案 + 强调清晰度** |

**记住：AI 图像生成不是设计工具，文字清晰度永远是一个权衡。**
